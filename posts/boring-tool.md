Every few months I rediscover that the unglamorous, decade-old tool is the one that actually ships the work. This is me writing it down so I stop forgetting.

## The pattern

It always goes the same way. There is a shiny new thing. The shiny new thing promises to make the boring thing obsolete. I spend a weekend with the shiny new thing. I come crawling back to the boring thing on Monday.

The boring tool has one quiet advantage the new one can never match: **time**. It has been broken in every way it can break, and someone already wrote down the fix.

## Not always

This isn&rsquo;t a blanket argument for never changing anything &mdash; that&rsquo;s how you end up maintaining a fax server in 2026. It&rsquo;s an argument for making the *new* thing earn its place, rather than assuming the old thing should be replaced just because it&rsquo;s old.
